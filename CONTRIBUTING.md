# Contributing

CatStays is governed by the central Codex Operating System in `vanessanz78/codex-operating-system`.

Before doing any work in this repository, every Codex session must read central `START_HERE.md` and follow the Operating System for:

- document reading order
- branch governance
- engineering standards
- architecture principles
- milestone workflow
- build verification
- UAT requirements
- cleanup procedures
- handoff requirements

This repository should not duplicate Operating System documentation. Keep CatStays documentation focused on project-specific product decisions, architecture, current sprint state, deployment notes, UAT evidence, and handoffs.

After the central Operating System startup read, continue with:

1. `CURRENT_SPRINT.md`
2. The latest root-level Architect Update, if one exists
3. `DECISION_LOG.md`
4. Any project files explicitly referenced by the sprint or decision documents

Every GitHub note, sprint note, decision note, PR note, and Replit handoff must include `Working ref: <main | branch name | commit SHA>`.
