# Customer Journey

Last reviewed: 2026-06-12

## Prospect Journey

A cattery customer should be able to:

1. Visit the public cattery website.
2. Understand rooms, prices, location, and care standards.
3. Request a booking.
4. Receive confirmation.
5. Pay a deposit or invoice.
6. Access their portal.
7. Manage cat details and future bookings.
8. Receive photo updates during the stay.

## Current Status

The public booking request flow exists. Customer portal routes also exist, including bookings and profile screens.

However, the customer portal is not yet fully production-wired. Some screens still use demo data and browser alert behavior.

## Target Portal Features

- Customer login.
- Customer profile.
- Cat profiles.
- Booking history.
- Upcoming bookings.
- Booking requests, changes, and cancellations.
- Invoices and payment links.
- Photo updates.
- Messages or update history.

## Experience Principles

- Customers should not need to learn software.
- Booking should feel calm and reassuring.
- Updates should feel personal but still owner-approved.
- Customer data must be isolated to that customer and cattery.
